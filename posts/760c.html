使用interrupt正确停止线程

中断策略

原理介绍：使用 interrupt 来通知，而不是强制。

最合理的中断策略是某种形式的线程级（Thread-Level） 取消操作或服务级（Service-Level）取消操作：尽快退出，在必要时进行清理，通知某个所有者该线程已经退出。此外还可以建立其它的中断策略，例如暂停服务或者重新开始服务。但对于那些包含非标准中断策略的线程或线程池，只能用于能知道这些策略的任务中。

中断可以理解为线程的一个标识位属性，它表示一个运行中的线程是否被其它线程进行了中断操作。中断就好比其它线程对该线程打了个招呼，其它线程通过调用该线程的 interrupt() 方法对其进行中断操作。

线程通过检查自身是否被中断来进行响应，线程通过方法 isInterrupted() 来进行判断是否被中断，也可以调用静态方法 Thread.interrupted() 对当前线程的中断标识位进行复位。如果该线程已经处于终结状态，即使该线程被中断过，在调用线程对象的 isInterrupted() 时依旧会返回 false。

从 Java 的 API 中可以看到，许多声明抛出 InterruptedException 的方法（例如 Thread.sleep(long mills) 方法），这些方法在抛出 IntercepedException 之前，Java 虚拟机会先将该线程的中断标识位清除，然后抛出 InterruptedException，此时调用 isInterrupted() 方法将返回 false。

普通情况停止线程

public class RightWayStopThreadWithoutSleep implements Runnable {

  @Override
  public void run() {
    int num = 0;
    while (num <= Integer.MAX_VALUE / 2 && !Thread.currentThread().isInterrupted()) {
      if (num % 10000 == 0) {
        System.out.println(num + " 是 10000 的倍数");
      }
      num++;
    }
    System.out.println("任务执行结束了");
  }

  public static void main(String[] args) throws InterruptedException {
    Thread t = new Thread(new RightWayStopThreadWithoutSleep());
    t.start();

    TimeUnit.SECONDS.sleep(1);
    t.interrupt();
  }
}

阻塞情况停止线程

public class RightWayStopThreadWithSleep {
  public static void main(String[] args) throws InterruptedException {
    Runnable runnable =
        () -> {
          int num = 0;
          while (num <= 300 && !Thread.currentThread().isInterrupted()) {
            if (num % 100 == 0) {
              System.out.println(num + " 是 100 的倍数");
            }
            num++;
          }
          try {
            TimeUnit.SECONDS.sleep(1); //第十八行
          } catch (InterruptedException e) {
            e.printStackTrace();
          }
        };

    Thread t = new Thread(runnable);
    t.start();

    TimeUnit.MILLISECONDS.sleep(500);
    t.interrupt();
  }
}

interrupt 失效

如果 while 里面放 try/catch，会导致中断失效。

public class CantInterrupt {
  public static void main(String[] args) throws InterruptedException {
    Runnable runnable =
        () -> {
          int num = 0;
          while (num <= 10000 && !Thread.currentThread().isInterrupted()) {
            if (num % 100 == 0) {
              System.out.println(num + " 是 100 的倍数");
            }

            num++;

            try {
              TimeUnit.MILLISECONDS.sleep(10);
            } catch (InterruptedException e) {
              //一旦响应中断，Java 虚拟机会先将该线程的中断标识位清除，所以导致中断失效
              e.printStackTrace();
            }

          }
        };

    Thread t = new Thread(runnable);
    t.start();

    TimeUnit.MILLISECONDS.sleep(500);
    t.interrupt();
  }
}

public class RightWayStopThreadInProd2 implements Runnable {

  @Override

    while (true) {
      if (Thread.currentThread().isInterrupted()) {
        System.out.println("Interrupted，程序运行结束");
        break;
      }
      System.out.println("go");
      reInterrupt();
    }
  }

  private void reInterrupt() {
    try {
      TimeUnit.SECONDS.sleep(1);

      Thread.currentThread().interrupt();

    t.start();

可以响应中断的方法

  • Object.wait() / wait(long) / wait(long, int)
  • Thread.sleep(long) / sleep(long, int)
  • Thread.join/ join(long) / join(long, int)
  • java.util.concurrent.BlockingQueue.take() / put(E)
  • java.util.concurrent.locks.Lock.lockInterruptibly()
  • java.util.concurrent.CountDownLatch.await()
  • java.util.concurrent.CyclicBarrier.wait()
  • java.util.concurrent.Exchanger.exchange(V)
  • java.nio.channels.InterruptibleChannel
  • java.nio.channel.Selector 的相关方法

补充

用 volatile 定义一个 boolean 变量来终止线程，在此线程陷入阻塞状态是无法终止时。

public class WrongWayVolatileCantStop {

  public static void main(String[] args) throws InterruptedException {
    CountDownLatch countDownLatch = new CountDownLatch(1);
    ArrayBlockingQueue storage = new ArrayBlockingQueue<>(10);
    Producer producer = new Producer(storage);
    Thread pThread = new Thread(producer);
    pThread.start();

    TimeUnit.SECONDS.sleep(1);

    Consumer consumer = new Consumer(storage, countDownLatch);
    Thread cThread = new Thread(consumer);
    cThread.start();

    countDownLatch.await();
    System.out.println("将要停止生产数据....");
    // 调用取消方法并没有停止，因为生产者线程处于阻塞状态
    // 为了更好的模拟，先演示1秒
    TimeUnit.SECONDS.sleep(1);
    producer.cancel();
  }
}

class Consumer implements Runnable {
  private BlockingQueue<Integer> storage;
  private CountDownLatch countDownLatch;

  public Consumer(BlockingQueue<Integer> storage, CountDownLatch countDownLatch) {
    this.storage = storage;
    this.countDownLatch = countDownLatch;
  }

  @Override
  public void run() {
    try {
      while (needConsumed()) {
        Integer take = storage.take();
        System.out.println("消费掉的数：" + take);
        TimeUnit.SECONDS.sleep(1);
      }
      System.out.println("消费者不需要更多数据了：");
      countDownLatch.countDown();
    } catch (InterruptedException e) {
      e.printStackTrace();
    }
  }

  int j = 0;
  private boolean needConsumed() {
    ++j;
    if(j == 20){
      return false;
    }else{
      return true;
    }
  }
}

class Producer implements Runnable {
  private volatile boolean canceled = false;
  private BlockingQueue<Integer> storage;

  public Producer(BlockingQueue<Integer> storage) {
    this.storage = storage;
  }

  @Override
  public void run() {
    int num = 0;
    try {
      while (num <= 100000 && !canceled) {
        if (num % 100 == 0) {
          storage.put(num);
          System.out.println(num + " 被添加到了队列");
        }
        ++num;
        TimeUnit.MILLISECONDS.sleep(1);
      }
    } catch (InterruptedException e) {
      e.printStackTrace();
    } finally {
      System.out.println("生产者停止运行");
    }
  }

  public void cancel() {
    canceled = true;
  }
}

Thread.interrupted() 方法，清除当前线程的状态，也就是 interrupet 置为 false。目标对象是当前线程（也就是这个方法目前在那个线程执行），不管在哪个对象调用本方法。
